Click Here to Play Monsters Inc. Maze

Description of Game

The concept behind this game is just a simple maze, but with a
Monsters Inc. theme so it is more enjoyable and interesting for
users. Basically, the main goal of the game is to get the character,
Mike Wazowski to collect as many coins as possible and exit the maze.
The amount of coins collected is basically the scoring system,
where the more coins collected, the higher the score. Randomness
occurs when the x-position of each of the coins are placed and set
within a given range. There were originally supposed to be three levels,
that increased in difficulty, but I only managed enough time to
create two levels. The first level is just a maze, but the second
level gets harder when the orange moving doors are added as obstacles.
It is more difficult because if you touched any of the orange blocks,
you would lose the game. However, if you can reach the exit
with no difficulties, then you would have finished and the game
would be over.

Depresso Cat

Summary of Depresso Cat

In this story, the user is put into the perspective of a little cat
on Christmas Eve, lonely and depressed. As a cat, you are given decisions
to make, ultimately leading to staying isolated, or finding happiness. Tragic
endings such as loneliness or even death can occur depending on what the user
chooses. But there are also happier endings, like finding a family and even
recovering your past memories. How the cat ends up is up to the user!

Development Process

My story was programmed on Python independently, by myself. Making a
flowchart was an incremental process because I broke the process into
separate pieces and started with the introduction, christmas_eve().
After, I typed out the next part to create separate scenes, like
wheel_encounter(). I created more events until I was satisfied. Each
scene was made into it's own function that was linked together to make
a flowing story. I also incorporated iterative processes into my
programming when I went through a cycle of writing the code, testing
it, and revising it, repeating this to solve any errors and improve my
coding. I made my story more engaging by having a time limit. I searched
up how to put a time limit on user inputs and added it into my program.
However, I did have some trouble because after the timer was added, the
program would stop working. I had to adapt to the situation by asking for
another user input, like 'Click space to continue.' Another problem was
when my coding skipped some functions. This was resolved when I checked
it and realized one function was spelled wrong.

Advantages of Abstraction

Organizing the scenes of my story is a form of procedural abstraction
since each of the scenes represented it's own function. For instance,
the function, home_sweet_home() is a method of procedural abstraction
because this scene with a bunch of different lines was created under one
name. It needed to be called more than one time, but with abstraction,
the advantage was that I didn't have to rewrite the entire code and I
could just call the one function name instead. This is also procedural
abstraction because I know that I want this function to be called after
meet_liams_fam() or after yEs_home_with_adelynn(), depending on which
path the user chooses, but how one function calls another function is
not entirely known. When I call a function, the program links scenes
together, but I don't know exactly how the program does it, meaning this
is procedural abstraction. Another form of procedural abstraction in my
story is the use of the typing() function. From a user's viewpoint, the
name typing() gives them a hint of what the function does, but not how
it does it. Similarly to the scene functions, there is an advantage to
procedural abstraction since it shortens the code by simplifying the
function. Instead of typing the code for typing() every time something
needs to be typed, I could just call the typing() function so the code
is simpler and easier for users to understand. In the end, there were
several uses of procedural abstraction within my code and story's program,
which adds many advantages, from being easy to code to being easy to read.
